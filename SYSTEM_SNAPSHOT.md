## 3. Environment Variables

The following environment variables are required for the backend to run. Optional variables for features like email and AI are noted.

| Variable | Description | Required? |
|---|---|:---:|
| `DATABASE_URL` | PostgreSQL connection string (e.g., `postgresql+asyncpg://...`) | **Yes** |
| `JWT_SECRET_KEY` | A long, random, secret string for signing JWTs | **Yes** |
| `JWT_ALGORITHM` | The algorithm to use for JWT signing | **Yes** (Default: `HS256`) |
| `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` | Expiration time for access tokens in minutes | **Yes** (Default: `1440`) |
| `CORS_ORIGINS` | Comma-separated list of allowed origins or `*` | **Yes** (Default: `*`) |
| `PORT` | The port the application server will listen on | **Yes** (Default: `8000`) |
| `APP_NAME` | Name of the application | No (Default: `RelationshipOS`) |
| `APP_VERSION` | Version of the application | No (Default: `1.0.0`) |
| `DEBUG` | Set to `true` to enable debug mode | No (Default: `false`) |
| `OPENAI_API_KEY` | API key for OpenAI to enable AI features | No (Optional) |
| `SMTP_HOST` | Hostname for the SMTP server for sending emails | No (Optional) |
| `SMTP_PORT` | Port for the SMTP server | No (Optional, Default: `587`) |
| `SMTP_USER` | Username for SMTP authentication | No (Optional) |
| `SMTP_PASS` | Password for SMTP authentication | No (Optional) |

## 4. Deployment & Run Instructions

The backend is designed to be deployed as a Docker container. The primary deployment target is Render.com, but it can be run on any platform that supports Docker or Python.

### Deploying to Render (Recommended)

The repository contains a `render.yaml` blueprint for one-click deployment.

1.  **Connect Repo:** In the Render Dashboard, create a new **Blueprint** and select the `relationshipos-api` GitHub repository.
2.  **Deploy:** Render will automatically read `render.yaml`, create the Web Service, and set the required environment variables.
3.  **Database:** The `render.yaml` points to a hardcoded Neon serverless PostgreSQL database. This is the production database and will be used automatically.
4.  **Live:** The service will be live at the URL provided by Render after the first build completes (approx. 3-5 minutes).

### Running Locally with Docker

1.  **Prerequisites:** Docker installed.
2.  **Environment:** Create a `.env` file in the root of the repository and set the required environment variables from the list above. For a local database, you can use a local PostgreSQL instance or the included SQLite fallback.
3.  **Build:** `docker build -t relationshipos-api .`
4.  **Run:** `docker run -p 8000:8000 --env-file .env relationshipos-api`
5.  **Access:** The API will be available at `http://localhost:8000`.

### Running Locally with Python

1.  **Prerequisites:** Python 3.11, `pip`.
2.  **Install Dependencies:** `pip install -r requirements.txt`
3.  **Environment:** Set the required environment variables in your shell or a `.env` file.
4.  **Run Server:** `uvicorn app.main:app --host 0.0.0.0 --port 8000`
5.  **Access:** The API will be available at `http://localhost:8000`.
